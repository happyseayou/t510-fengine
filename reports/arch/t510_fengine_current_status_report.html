临时实现与边界

当前为了 bring-up 临时加入

  • LABtcxo_10mhz + free_run 路径，用于无 PPS/无外部 10 MHz 环境。
  • LABADC0-only active mask 0x0001，避免当前只接一根线时等待所有 RFDC ports。
  • LABDAC tone source，作为 ADC0-DAC0 闭环激励。
  • LAB1024 点 debug FFT 与 BRAM readout，用于 PYNQ 直接看波形和频谱。
  • LABpacket TX AXIS 当前不接 CMAC，只保留内部活动观测。

设计期望但尚未完成

  • TARGET8 路 complex baseband 全输入验收。
  • TARGET正式 4096-channel、4 taps 起步的 PFB/FFT channelizer。
  • TARGETSPEC/TIME 包通过 CMAC/QSFP28 100G 输出。
  • TARGETSnapshot DMA 写 DDR，由 PYNQ 拉取大块时域数据。
  • TARGET外部 10 MHz/PPS 多板同步验收。

已知风险与下一步

最需要清理的技术债

DEBUG_STATUS=0x6 目前不影响读取，bit2 done 已经确认 buffer 有效；但 bit1 error 不应该在“可用 capture”上常亮。下一步应检查 fft_debug_observer 内部对 XFFT event 或 backpressure 的错误判定，把它改成真正的 fatal-only 标志。

建议执行顺序

  1. 修正 fft_debug_observer 的 error bit，保持 done/error 语义干净。
  2. 用 Jupyter notebook 改变 DAC_TONE_PHASE_STEP，确认 FFT peak bin 随 tone 移动。
  3. 把 RFDC adapter 从“每 port 每 beat 取 1 个 16-bit sample”扩展到更接近真实吞吐的多样本路径。
  4. 将 debug FFT 的观测路径和正式 channelizer 设计解耦，开始接入 4096-channel SPEC 路径。
  5. 接 CMAC 前先完成 UDP/IP framing 的仿真向量和 host-side packet checker。
  6. 外部 PPS/10 MHz 条件具备后，切回 external_pps，验证 arm 后下一次 PPS 进入 streaming。
