T510 Board Agent API

Stage 30 · stateless Rust service · OpenAPI 3.1 · Agent version 0.1.0

开发网络安全模式：security_mode=none。服务不校验 token、allowlist 或 mTLS，不启用 CORS。请仅放在受控实验室局域网中。

职责边界

Agent 只做配置和控制，并在请求时读取一次累计寄存器快照。它不抓取波形/频谱，不做 FFT、WebSocket、packet capture 或后台轮询，也不计算 packet rate/drop rate（does not calculate packet rates）。Center Hub 应保存相邻 GET /api/v1/status 快照并计算速率。

每个硬件请求同步启动一次固定 Python helper。Agent 同一时刻只接受一个自身发起的硬件请求；冲突返回 409 HARDWARE_BUSY。Agent 不管理 Jupyter，也不能阻止 notebook 同时写硬件。

发现与文档

方法路径说明
GET/302 跳转到本页。
GET/api/help自包含说明页。
GET/api/openapi.jsonOpenAPI 3.1 JSON。
GET/health/liveRust 进程存活，不访问 FPGA。
GET/health/ready检查本地配置、helper、Python 和 catalog 文件，不访问 FPGA。
GET/api/v1/infodevice UID、hostname、machine-id、管理 MAC/IP、内存、版本和安全模式。
GET/api/v1/capabilities明确支持和不支持的能力。
GET/api/v1/bitstreams可用的命名 bitstream ID、SHA、core version 和 profile；不公开任意路径入口。

硬件接口

方法路径行为
GET/api/v1/status一次性读取 core、board ID、streaming、时钟/PPS/QSFP、累计 packet/drop/TX counters、error flags 和 sample0。
POST/api/v1/configure加载 catalog bitstream，配置 profile/source/24 endpoints/board ID；完成后保持停流。重复调用会真实重配。
POST/api/v1/start校验 expected_board_id 后立即启动。
POST/api/v1/stop无条件安全停止并验证 counters 稳定；无需请求体。
POST/api/v1/reset校验 expected_board_id，先安全停止再 reset。
PUT/api/v1/dac校验 board ID，短暂 mute，原子更新完整八路 DAC，phase commit，恢复 enable mask 并回读。

配置请求

endpoints 必须恰好 24 项，ID 0..23 各一次；0..7 为 TIME，8..23 为 SPEC。profile 的 mode 必须与 enable mask 一致。IP/MAC 必须是确定值，板端不做 DNS/ARP 推断。

curl -sS -X POST http://192.168.100.117:8010/api/v1/configure \
  -H 'Content-Type: application/json' \
  --data @config/stage30/configure.example.json | jq

启动、停止和 reset：

curl -sS -X POST http://192.168.100.117:8010/api/v1/start \
  -H 'Content-Type: application/json' -d '{"expected_board_id":1}' | jq
curl -sS -X POST http://192.168.100.117:8010/api/v1/stop | jq
curl -sS -X POST http://192.168.100.117:8010/api/v1/reset \
  -H 'Content-Type: application/json' -d '{"expected_board_id":1}' | jq

DAC 请求

curl -sS -X PUT http://192.168.100.117:8010/api/v1/dac \
  -H 'Content-Type: application/json' \
  --data @config/stage30/dac.example.json | jq

请求必须包含 expected_board_id、center_mhz 和 channel 0..7 的完整八路 enabled、rf_frequency_mhz、amplitude_percent、phase_deg。

响应与错误

成功响应格式：{"request_id":"...","result":{...}}。错误格式包含稳定 code：

{
  "request_id": "t510-...",
  "code": "HARDWARE_BUSY",
  "message": "another Agent hardware request is running",
  "details": {}
}

常见 HTTP 状态：400 schema/JSON，404 unknown bitstream，409 busy/board-state conflict，503 Python/PYNQ/hardware unavailable，504 Python timeout。configure 默认 180 秒，其余硬件请求默认 10 秒。
